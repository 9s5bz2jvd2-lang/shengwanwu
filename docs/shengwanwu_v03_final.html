🌿 缘起

世间万般学问，散落于千万论文之中。如夜空繁星，各自璀璨，却鲜有人将其连缀成图。

《生万物论》所求者，非替代人之思考，乃为研究者提供一副"自动望远镜"——从浩瀚文献中，蒸馏知识、编织图谱、发现矛盾、生发假说。

一如道家所言：从混沌（原始论文库）中生出秩序（知识图谱），秩序中孕育变化（假说种子），变化中绽放万相（可行动的研究问题）。

道
原始论文库 · 道之混沌
arXiv PDF、书籍、数据库——未加工的知识矿藏。如"道"之初，浑然未分。
一
蒸馏成库 · 道生一
将论文全文蒸馏为结构化知识节点（method/finding/concept/limitation），如"一"之始立——从混沌中提取秩序。
全文提取 PyMuPDF trafilatura JSONL
二
图谱成网 · 一生二
知识节点之间，以共享标签自动建立连接（edges），编织成跨论文知识图谱。在图谱中检测六种模式：方法缺失、概念矛盾、跨域桥梁、共享关键词、传递断链、枢纽共现。如"二"之对立——阴与阳，同与异，连接与断裂。
共享tag→edges 6种检测模式 v0.2 scanner
三
语义重构 · 二生三
三层语义重构架构，将机械的图谱模式转化为有意义的科学假说：
Layer 1 — 图谱模式检测（v0.2）→ 原始假说候选
Layer 2 — 语义上下文组装 → 假说上下文包（蒸馏全文 + 模式语义 + 关联边）
Layer 3 — LLM语义重构 → 结构化研究问题（含实验方案 + 证伪条件）
类型特定prompt模板 并行分神处理 DeepSeek/Claude Code
万
万物生发 · 三生万物
从去重、排序后的假说池中，精选 Top-N 可行动研究问题。每条假说包含：可证伪声明、实验方案、评估指标、基线对比、预期瓶颈、新颖性/可行性/影响力评分。如"万物"之繁盛——从简单规则中，绽放出无尽的研究可能。
去重 Grounding check 优先级排序 Top-20

💭 设计哲学

由库成论法：不依赖预训练知识，一切假说从蒸馏的论文全文中生成。每条假说可追溯到源文献。

六眼旁观：模式检测如六只眼睛，分别审视方法缺失、概念矛盾、跨域桥梁、关键词暗连、传递断裂、枢纽汇聚。多视角才能见全局。

论证门与验证门：假说不经论证不成形，不经验证不出口。生成可追溯、可证伪、可验证的候选，不自动证明理论。

低功耗之道：确定性工作交脚本，判断性工作交LLM。人审稿拆小单元。失败时修最小失败层，不整炉重炼。

🔮 未来展望

v0.4 规划：引入 embedding-based 语义聚类（替代 tag-based），实现更精准的假说去重与主题归并。

开放假说：研究者可输入自己的假说，引擎自动在图谱中寻找支持/反对证据。

回流机制：验证结果回流知识库，形成"蒸馏→假说→实验→回流"的闭环。
